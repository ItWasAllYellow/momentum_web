## Slide 2: 기획 의도 (Project Intent)
- **내용**:
  - **문제점**: 금융 정보의 비대칭성. 전문 애널리스트 리포트는 일반인이 이해하기 어렵고, 뉴스는 너무 많음.
  - **목표**: "금융 비서의 대중화".
  - **핵심 가치**: 
    1. **쉬운 접근 (Easy)**: 초보자도 이해하기 쉬운 거장(Guru)의 조언.
    2. **심층 분석 (Expert)**: 리포트 행간의 의미(Tone)를 읽어내는 AI.
- **레이아웃/디자인**:
  - **좌측**: 텍스트 (문제점 -> 목표 -> 해결책).
  - **우측**: 복잡한 주식 차트 위에 '?'가 떠있는 이미지 vs AI 로봇이 웃으며 설명해주는 이미지 대비.

---

## Slide 3: 솔루션 - Momentum (Solution Overview)
- **내용**:
  - **Dual Mode Approach**: 사용자의 숙련도에 따른 맞춤형 모드 제공.
  - **Easy Mode**: 직관적인 포트폴리오 관리 + AI 대가(버핏 등)의 코멘트.
  - **Expert Mode**: 350개 종목 데이터 + 애널리스트 톤 변화 감지 + 키워드 뉴스 필터링.
- **레이아웃/디자인**:
  - 화면을 반으로 나누어(Split Screen) 표현.
  - **왼쪽(Easy)**: 밝고 따뜻한 색감, 심플한 UI 예시.
  - **오른쪽(Expert)**: 전문적인 다크 모드, 데이터 밀도가 높은 UI 예시.

---

## Slide 4: 핵심 기능 1 - Guru Analysis (Easy Mode)
- **내용**:
  - 사용자의 포트폴리오를 유명 투자자(워렌 버핏, 피터 린치)의 관점에서 진단.
  - 단순 수익률 계산을 넘어, 뉴스/기술적 지표를 종합한 AI 리포트 생성.
  - **기술적 특징**: LLM(Gemini)에 투자 철학 프롬프트 주입.
- **레이아웃/디자인**:
  - **상단**: 기능 설명 한 줄.
  - **하단 좌측**: 실제 구현 화면 캡처 (Guru Analysis 결과창).
  - **하단 우측**: 워렌 버핏 아이콘이 말풍선으로 조언하는 그래픽.

---

## Slide 5: 핵심 기능 2 - Tone Change Analysis (Expert Mode)
- **내용**:
  - **What**: 증권사 리포트 텍스트 분석.
  - **Why**: "매수" 일색인 리포트 속에서 미묘한 어조 변화(긍정->중립, 어휘 변화)를 감지하여 실제 뷰를 파악.
  - **How**: 과거 리포트 vs 최신 리포트 비교 분석 (Report-to-Report).
- **레이아웃/디자인**:
  - **중앙**: 흐름도 (Flowchart).
    - `[과거 리포트] + [최신 리포트] -> [AI 분석 엔진] -> [Tone Change Score & Insight]`
  - **예시**: "목표가 유지하나 우려 상존" 같은 텍스트를 AI가 빨간 밑줄로 강조하는 이미지.

## Slide 6: 기술 스택 (Tech Stack)
- **내용**:
  - **Frontend**: React 19, Vite, Recharts (데이터 시각화).
  - **Backend**: Python FastAPI, Pandas (데이터 처리), Google Gemini API (LLM).
  - **Data**: CSV(주가), JSON(뉴스), Text Files(리포트).
- **레이아웃/디자인**:
  - 3단 구성 (Frontend / Backend / AI & Data).
  - 각 영역에 기술 로고(React, Python, Gemini) 아이콘 배치.
  - 데이터 흐름 화살표로 연결.

## Slide 7: 시스템 아키텍처 (System Architecture)
- **내용**:
  - **Client**: 사용자 입력 -> API 요청.
  - **Server**: 
    1. 데이터 로더 (CSV/JSON 메모리 적재).
    2. 데이터 서비스 (수익률, 기술적 지표 계산).
    3. AI 서비스 (Gemini 프롬프트 엔지니어링 및 호출).
  - **External**: Google Gemini Pro.
- **레이아웃/디자인**:
  - 다이어그램 (Block Diagram).
  - 중앙의 FastAPI 서버를 중심으로 Client와 AI Model이 연결된 허브앤스포크 형태 추천.

## Slide 8: 시연 시나리오 (Demo Scenario)
- **내용**:
